# Shorts Stage Gate And Reset Policy

Use this reference with `youtube_shorts_production_v1/SKILL.md` when deciding whether a Cascade Effects short may advance, reset, or route back to a specialist stage.

## Challenger-First Restart Scope

- Challenger is the only active restart target until a DP-approved workflow scope manifest opens another episode.
- All existing visual constraints, old renders, model experiments, packaging rules, keeper registry lessons, casebook lessons, old manifests, and previous episode artifacts are legacy by default.
- Old Challenger, Hyatt, Therac, and 737 constraints do not become active automatically.
- New visual production must use a new scoped Challenger short root and must not start from old `challenger_short_v1`, `challenger_short_v2_exact`, `challenger_short_v3_trimmed`, or restart visual folders.
- Visual context may hydrate only from scoped active roots, exact whitelisted source/audio files, and exact DP-approved imports listed in the workflow scope manifest.
- `/archive/`, `/archives/`, `/experiments/`, `/legacy/`, `/retired/`, `/midjourney/`, `/proof_stills/`, diagnostic outputs, mixed-review outputs, old renders, old manifests, old casebook/keeper-derived constraints, and old model experiment rules are blocked unless an exact file is DP-imported.

## Stage Gate Detail

1. `script`: require a source writer packet, fact-checked long-form audio-only script, long-form fact-check report, derived 60-second Short script, narration map, cue ranges, timing intent, proof the Short introduces no unsupported new claims, and a closing Signature Consequence Motif unless the coordinator records an explicit waiver. Lane-specific variants require `brand_motif_status: variant`, `motif_family`, and exact `motif_text`. The narration map is timing/causality input only; it is not the canonical visual beat structure.
2. `audio`: require a packaged WAV, QA transcript/timing evidence, `audio_package.json`, voice/model/provider/profile provenance, package and transcript hashes, script-locked caption handoff fields, and Signature Consequence Motif cadence QA. Audio must be `keep`, match the active Shorts voice profile, and record `brand_motif_status: present|variant|waived` plus `ending_cadence_read: pass` before visual stages advance. `variant` requires `motif_family` and exact `motif_text`; `waived` requires a coordinator-owned `motif_waiver_reason`.
3. `DP scope/research brief`: require a DP-approved workflow scope manifest and DP research brief. The brief must ask which imported artifacts can carry the short most engagingly before the visual beatmap is locked.
4. `visual research packet`: require a beat-zone reference packet from scoped source/audio files, the narration map, the DP research brief, and the approved audio transcript. If the scope manifest imports motion-reference video, require a DP-owned `archival footage review` before the beatmap is locked. That review must use `archival role: hybrid`, `hygiene rule: strict clean`, `analog look: selective`, and `source breadth: 1-3 primary archival videos plus up to 2 backups`, and each archival source must carry an exact local media path plus original source URL or origin note. Record evidence for proposed constraints, source/reference paths, physical scale, custody/access, literal-vs-representational status, prompt risks, unresolved visual research gaps, ranked artifacts, preferred archival clips, and source-derived reanimation suitability. The packet is evidence, not an active constraint ledger. It must record actual artifact ranking with `artifact_priority_rank`, `artifact_engagement_score`, `mechanism_visibility_score`, `motion_potential_score`, `downstream_coverage_role`, `scene_family`, and `artifact_selection_rationale`, plus `preferred_clip_ids`, `backup_clip_ids`, `footage_family_id`, `archive_reference_mode`, `texture_influence`, `source_anchor_id`, `recognizable_source_anchor`, `source_anchor_paths_or_urls`, `anchor_preservation_rule`, `carrier_mode`, `reanimation_strategy`, `anchor_drift_fail_conditions`, and `nonliteral_exception_approved`.
5. `production model decision`: require a coordinator-owned lane decision after visual research and before visual beatmap. Valid lanes are `source_led_motion`, `source_derived_reanimation`, `sourced_stills`, `generated_stills`, and `hybrid`. The decision must name 2-4 hero source families, explain why the lane is the cheapest viable path to an engaging Short, and state whether generated stills are expected or exception-only.
6. `visual beatmap`: require a DP-owned canonical visual beatmap derived from the ranked research artifacts and the production model decision. Challenger uses `9 beats + short pre-beat`. The beatmap must cite preferred and backup artifact ids for each beat zone and, when archival motion is in scope, preferred and backup clip ids plus a footage family id. Multiple narration lines may map to one strong archival footage family.
7. `DP shot planning`: require `shot_plan_v2` derived from the approved visual beatmap. Every shot must reference `visual_beat_id`. When archival motion is in scope, every shot must also reference the relevant archival clip ids or footage family. The shot plan must include an `editorial_spine_brief` naming hero source families, source-family budget, continuity vector, and any subject/event coverage that should beat literal narration coverage.
8. `episode constraint ledger`: require DP approval before render authorization or still generation. Classify all proposed and legacy constraints as `active`, `conditional`, `legacy_reference`, or `retired`. Only `active` and currently satisfied `conditional` constraints may control prompts.
9. `render authorization check`: require a coordinator-owned check before still or motion generation. For each shot, record whether an approved source clip/still already solves it, source-derived reanimation is better, or generated stills are actually necessary. Set `render_authorization_read: pass|tighten|reject`.
10. `stills/keyframe contact sheet`: require shot-level clean candidates from approved archival still exports, restored research assets, or clean composites/crops built from restored source assets before any review styling is applied. This stage is carrier and keyframe selection, not automatic AI still generation. If sourced archival frames already clear the shot, review those assets directly instead of generating replacement stills. When source-derived archival reanimation is active, the contact sheet must show the full-frame clean source frame and, when relevant, proposed start/end keyframes for each motion handle. Build the sheet directly from raw full-frame clean candidates and keep beat info on the sheet itself as minimal plain-text labels only. Borders, blurred fills, padded presentation tiles, or other review styling are not the default. Record review notes and current best stills, keyframes, or named gaps. Each reviewed still/keyframe candidate must record `generation_source_paths`, `clean_candidate_path`, `contact_sheet_tile_path`, `generation_surface_clean`, `carrier_mode`, `reanimation_strategy`, `anchor_match`, `coverage_read`, `variety_read`, `engagement_read`, and `hygiene_read`.
11. `stills video proof`: assemble selected stills against approved audio when still/keyframe pacing needs review. This proof may be diagnostic and is not final, but it must render from `clean_candidate_path`, not any styled review surface. A sourced still is a valid proof carrier; generated stills are not required when the sourced asset already solves the shot.
12. `motion contact sheet`: render or trim motion candidates from approved source surfaces and source spans. Create a real motion-test grid; do not rebuild it by harvesting replacement frames or swapping in newly chosen imagery after the still/keyframe gate. Valid motion strategies are `direct_source_clip`, `source_derived_reanimation`, and `still_driven_i2v` when approved upstream. Group the motion contact sheet by beat, motion strategy, source still/keyframes or source clip, motion pipeline, prompt variant, and seed. Motion candidates in this stage must target at least `5.0` seconds, even when the underlying beat is shorter. Review every completed candidate by actual clip playback plus dense frame sampling before any handoff. For launchpad, plume, vehicle, rigid-geometry, historical machinery, explosion, human/tableau, clinical, or other high-risk beats, require frame-by-frame or near-frame-by-frame review rather than sparse thumbnails alone. Any backend-generated or source-retained audio must be stripped before review or proof assembly.
13. `shot_timing_edl`: require one row per actual story shot before motion video proof. Each row must record source span, intended and actual duration, parent beat, continuity vector, and `no_internal_cut_read`. Beat handles and beat rollups are metadata only; the EDL is the proof-edit authority.
14. `historical_signal_texture`: house CRT substep after motion contact sheet winners are selected and before `motion video proof`. It applies to `keep`, no-audio, source-clean story clips with `texture_influence: house_crt` unless the coordinator records an explicit waiver. It must use `era_1980s_broadcast_crt_v1` at `visible_but_premium`, remain full-bleed with no TV frame/matte/border/reduced image area, and pass visibility, house CRT consistency, Challenger-style signal-interruption review when cut interruption is used, and YouTube survivability review before entering a keeper proof.
15. `motion video proof`: assemble selected `keep` winners against approved audio from the approved `shot_timing_edl`, generate a beat sheet beside the proof, record `beat_sheet_path` in the proof manifest, and label unresolved cuts as `mixed review short`. A timed A/B comparison proof is diagnostic unless selected winners are separately reviewed as `keep`. Proof review must pass `proof_assembled_from_shot_timing_edl`, `contact_sheet_to_proof_parity_read`, `hidden_cut_read`, and `story_shot_duration_read`. If a proof uses textured archival clips, record the recipe, textured path, source lane, YouTube proxy path, and texture reads in the proof manifest.
16. `video final`: route to the final-export skill with the approved motion proof, approved audio, `caption_text_source_path` from locked script text or script-locked captions, `caption_timing_source_path` from ASR/WhisperX timing evidence, proof review note, and caption style preset. Active Shorts default to the music registry's `paper_architecture_theme_v1` as `music_policy: canonical_default`; apply it only at this stage as `motif_outro_mix` after the voice/caption timing is locked. If the coordinator records `music_policy: waived` or `music_policy: alternate_approved`, the final-export request must carry the waiver reason or registry-backed alternate track.
17. `publish package`: require a package-level `target: youtube_shorts` manifest, upload MP4, title, description, tags, script-locked SRT, cover frame, final review, final-export manifest, source-final keeper fields, caption text/timing provenance fields, and declared SHA-256 values. Run `publish-package-check`; validation issues block upload, while rights/claims/music warnings block public release until Studio review.
18. `unlisted review upload`: route to the publish skill. Upload only as `--privacy unlisted` after action-time confirmation, write an upload receipt beside the package, verify status with `publish-package-status`, and keep public release manual. Delete mistaken review uploads only with exact video-id confirmation and a delete receipt; if the OAuth token lacks delete scope, route deletion through YouTube Studio.
19. `keeper lesson capsule`: after a publishable `keeper short`, record reusable lessons only: lane choice, winning source families, failed failure modes, final texture/audio/caption/publish decisions, and whether each lesson is global policy, episode-local, or diagnostic history.

## Promotion And Manifest Rules

- Use only `keep`, `tighten`, `diagnostic only`, and `reject`.
- Reel/final classes are `keeper short` and `mixed review short`.
- Only `keep` audio advances into visual work.
- Active Shorts must use `youtube_shorts_mike_challenger_match_v1` unless the coordinator explicitly promotes a new active profile in the audio lane registry.
- Same voice ID with the wrong model/profile is a blocker; `eleven_v3` Mike voice lanes are comparison or diagnostic only until promoted.
- Active short manifests, proof manifests, and final export manifests must carry `short_audio_package_path`, `expected_voice_profile_id`, `audio_package_sha256`, `audio_disposition`, `caption_text_source_path`, `caption_timing_source_path`, and transcript hash fields for audio provenance.
- `audio_path` must match the audio package `packaged_path`; `caption_text_source_path` must be locked script text or script-locked caption output, while the audio package QA transcript/WhisperX output may be used only as timing evidence.
- The caption model is `script_locked_canonical_text_timing_from_asr_v1`; `.diarized.*`, WhisperX, raw ASR, VTT, and SRT timing files are blocked as caption word sources unless already script-locked and QA-passing.
- Every active YouTube Short must close on the Cascade Effects Signature Consequence Motif unless a coordinator-owned waiver is recorded. The default motif is `Small causes, massive consequences.`
- Lane-specific variants are valid when they are registered as motif families. The approved `Mystery That Has Receipts` variant is `motif_family: mystery_that_has_receipts`, `motif_text: The mystery fades. The receipts remain.`, and `brand_motif_status: variant`.
- Audio and downstream proof/final manifests must preserve `brand_motif_status`, `motif_family` when applicable, `motif_text`, `motif_waiver_reason` when applicable, and `ending_cadence_read`.
- A correct transcript with a questioning, upward, cut-off, or unresolved final cadence is `tighten` or `reject`, not `keep`.
- Use Challenger `ending_cadence_pass_01` as the current reference for acceptable terminal delivery: calm, declarative, downward or flat resolution, and enough post-line silence to feel complete.
- Active Shorts default to `music_policy: canonical_default`, `music_track_registry_path: /Users/mike/CascadeEffects/packages/production-registry/shorts/music_track_registry.json`, and `music_track_id: paper_architecture_theme_v1`. Waived music requires a non-empty `music_waiver_reason`; alternate music requires `music_policy: alternate_approved` and a registry-backed `music_track_id`.
- Final music beds and theme outros must support the Signature Consequence Motif rather than competing with it. The body bed stays under narration, the outro may enter quietly before the final line finishes, and the audible outro ramp should begin after the motif resolves. If the outro cannot resolve cleanly before the final frame, start with a short final-frame hold of about `0.5-0.75` seconds; when the registered track policy requires completion, extend visual duration long enough for the outro to finish. If a long extension creates visible freeze and clean continuation footage exists, use a no-audio `source_motion_tail` instead of cloned-frame padding.
- Final music mixes must record `music_track_registry_path`, `music_track_id`, `music_policy`, `music_rights_check_status`, `motif_outro_mix_used`, loop/outro source paths and hashes, timing, volume/ramp values, visual extension mode, `motif_music_bed_read`, `outro_completion_read`, `final_mix_peak_db`, final-frame hold, source tail path/span when used, residual cloned hold, outro duration used, and cutoff seconds. A mix that masks the motif, makes the ending feel like a question, clips, visibly freezes when motion continuation was available, or cuts off the outro is `tighten` or `reject`.
- Publishing is package-based only. Use `publish-package-check`, `publish-package-upload --privacy unlisted`, `publish-package-status`, and exactly confirmed `publish-package-delete`; never use deprecated episode-level publish commands for Shorts.
- YouTube public release remains manual and must follow Studio checks for copyright/Content ID, Paper Architecture music, altered-content disclosure, captions, cover frame, audience, metadata, and visibility.
- Burned-in captions cannot be disabled on YouTube. Uploaded SRT or auto captions should be reviewed or removed if they duplicate the burned-in caption layer.
- Active short manifests live directly under `/Users/mike/CascadeEffects/packages/media-pipeline/viz/references/episodes/<episode>/shorts/`.
- Experimental or comparison-only manifests are legacy/reference-only during the restart. `shorts/experiments/` is blocked unless the DP scope manifest opens an exact experimental path for a diagnostic task.
- Long-form audio packages are cataloged only and must not enter Shorts manifests, proofs, or finals.
- Active visual manifests must record `workflow_scope_manifest_path`, `dp_research_brief_path`, `episode_constraint_ledger_path`, and exact DP imports used.
- Active visual manifests must record `production_model_decision_path`, `production_model_lane`, `hero_source_family_ids`, `render_authorization_read`, and `shot_timing_edl_path` once those gates exist.

## Visual Gate Rules

- The narration map must precede `visual research packet`.
- `DP scope/research brief` must precede `visual research packet`.
- `visual research packet` must precede `visual beatmap`.
- `visual beatmap` must precede `shot_plan_v2`.
- `shot_plan_v2` must precede `episode constraint ledger`.
- A DP-approved active episode constraint ledger and render authorization check must precede `stills/keyframe contact sheet` or motion generation.
- Challenger visual research must rank artifacts by downstream scene strength. Scene-led artifacts are first choice, mechanism inserts are second choice, and flat paperwork, folders, trays, or admin-detail props are exception-only.
- The DP must apply the `engaging and visually stimulating` thesis during artifact selection because research artifacts drive downstream still and motion choices.
- The production model decision must precede the visual beatmap. If the decision is missing, stale, cannot name hero source families, or chooses generated stills without explaining why sourced assets cannot solve the shot, block beatmap lock.
- When archival motion is in scope, complete the archival footage review before the visual beatmap, keep the archival source pool to `1-3` primary videos plus up to `2` backups, and require exact local media paths plus original source URLs or origin notes.
- Under `strict clean`, reject any archival span with a logo, stinger, lower-third, watermark, burned caption, end card, split screen, or channel bug.
- One strong archival footage family may satisfy multiple narration lines. Do not force literal one-line-one-image mapping.
- Each beat or shot must declare a recognizable source anchor before still generation unless DP has explicitly approved `nonliteral_exception_approved: true`.
- Still prompts must use the approved visual research packet, visual beatmap, and only active ledger constraints.
- If the packet is missing, stale, or too vague to constrain the beat, route back to visual research before rendering stills.
- If the visual beatmap is missing, stale, or still narration-derived instead of research-driven, route back to DP and visual research before rendering stills.
- If scope manifest, DP research brief, visual research evidence, or active ledger is missing, block still generation.
- Approved sourced carriers are allowed when they are explicitly recorded as `carrier_mode: sourced` or `hybrid`; do not treat them as illegal reference copying.
- If an approved sourced still or archival clip already solves the shot, treat generated stills as exception-only. Do not regenerate a source-complete shot in Flux just to hand it to motion.
- If render authorization says a sourced still/clip or source-derived reanimation solves a shot, generated stills are blocked for that shot unless the DP records a later override.
- If archival motion has the right visual language but not the required duration or beat emphasis, use `source_derived_reanimation`: clean reviewed source span plus same-camera keyframes plus an audited no-audio LTX handle.
- Do not keyframe across different camera angles, scenes, or footage families. Cross-angle interpolation is a `reject`.
- Source-derived reanimation candidates must carry `source_clip_id`, `source_clip_path`, `source_span_in`, `source_span_out`, `start_keyframe_path`, `end_keyframe_path`, `same_camera_span_confirmed`, `keyframe_crop_consistency`, `raw_motion_path`, `normalized_no_audio_path`, `audio_stream_read`, `frame_audit_paths`, `source_motion_alignment_read`, and `archival_fidelity_read`.
- `generation_source_paths`, `clean_candidate_path`, and the raw `contact_sheet_tile_path` are authoritative still surfaces. Optional styled review plates, compiled contact-sheet images, and any path under `stills/contact_sheet_pass_*/raw/` are non-authoritative review artifacts.
- If a case uses `source_kind: research_board_composite`, the board composite is selection-only and may not enter proof or generation until the case resolves to real clean source files.
- For Challenger, literal technical/evidentiary coverage stays support-only: one insert-heavy beat family or two insert shots total unless the DP records an override.
- `texture_influence` values are `none|house_crt`; use `house_crt` for active story clips entering proof/final unless explicitly waived.
- The registered active house texture is `/Users/mike/CascadeEffects/archive/season-01-reference/legacy-style-packages/style-packages/source_preserving_documentary_v1/archival_signal_texture_recipes.json` profile `era_1980s_broadcast_crt_v1` at `visible_but_premium`. Do not choose film, institutional-video, 1990s-news, 2000s-web, or mobile-video profiles by episode period/source medium.
- Any textured clip entering proof or final must record `historical_signal_profile_id`, `texture_source_lane`, `texture_applied_path`, `signal_texture_strength`, `texture_visibility_read`, `house_crt_texture_read`, `signal_interruption_read`, `youtube_survival_proxy_path`, `youtube_survival_read`, `compression_artifact_read`, `detail_survival_read`, and `historical_signal_texture_read`.
- Reject or downgrade texture if it causes `texture_invisible_no_reviewable_historical_signal`, `historical_signal_overpowering_subject`, `compression_shimmer_or_mud`, `historical_signal_cheese`, `detail_loss_from_signal_texture`, `signal_interruption_masks_story_footage`, fake text/logo/caption contamination, face/detail loss, or machinery/vehicle/plume/debris readability loss. Fallback order is lower CRT strength, then `conservative_clean`, then `baseline`, then better source search.
- Visual research packets are constraint artifacts only; do not treat reference images, logos, archival document text, or artist style names as prompt targets to copy.
- Treat casebook, keeper registry, subject matrix, old style docs, and model experiment notes as scoped references only; they cannot activate constraints unless the active ledger does so.
- If a source path is outside the workflow scope and has no exact DP import record, it cannot be used.
- If a contact-sheet still no longer reads as its named research anchor and instead reads like a generic evidence-room prop, record `anchor_match: reject`, classify the failure as `anonymous evidence-room drift`, and route back upstream.
- If a beat uses `coverage_class: object_cluster`, require a multi-object tableau with at least two approved artifacts visible in-frame; a single isolated object is `tighten` or `reject`.
- If a still is anchor-correct but still weak, repetitive, or low-energy, record `coverage_read`, `variety_read`, or `engagement_read` as `tighten` or `reject` and route back upstream.
- If a sourced or hybrid output inherits logos, stingers, lower-thirds, channel bugs, or burned captions from archival footage, record `hygiene_read: reject`, classify the failure as `absorbed archival contamination`, and route back upstream.
- If a styled contact plate, blurred-fill composite, framed review artifact, or compiled contact sheet enters generation or proof as if it were a clean source still, record `generation_surface_read: reject`, classify the failure as `review-surface contamination`, and route back upstream.
- Motion contact-sheet review must record `review_method`, `temporal_coherence_read`, `physical_plausibility_read`, and `source_motion_alignment_read` for each surfaced candidate. A motion candidate is not review-complete until those reads are recorded from actual clip inspection.
- Hard motion blockers include source-defying event invention, impossible flame/smoke/plume behavior, rigid-structure drift that violates the source shot, and any motion that turns a static historical setup into a different event than the source still or approved source clip implies.
- If a launchpad-, vehicle-, or machinery-led beat fails because the model invents ignition, plume, liftoff, structural drift, or other source-defying motion, route that beat to direct sourced archival motion or a new still/carrier decision rather than spending another pass on the same still-driven failure mode.
- If a human, clinical, control-room, courtroom, or office beat fails because reanimation invents new people, role changes, sit/stand action, medical action, facial identity drift, or room-layout drift, route back to source span/keyframe selection instead of prompt tightening.
- `stills/keyframe contact sheet` must precede `stills video proof` when a still proof is needed.
- Approved source clips, source frames, keyframes, or sourced stills may enter `motion contact sheet` when render authorization allows them; a generated still is not required.
- `motion contact sheet` must precede `motion video proof`.
- `shot_timing_edl` must precede `motion video proof`.
- `motion video proof` must precede `video final`.
- `video final` must precede `publish package`.
- `publish-package-check` must pass before unlisted review upload.
- Only `keep` source surfaces, keyframes, source clips, or stills are eligible for motion.
- Only `keep` motion clips are eligible for final.
- For hard or motion-brittle beats, require a primary still plus one alternate motion-ready still before the motion candidate matrix. Hard beats include `human/tableau`, strict historical geometry, locked-context shots, or any beat where motion already exposed a source-baked defect. Do not require two stills for every beat.
- No diagnostic placeholders in a `keeper short` or `video final`.
- No silent mutation of approved long-form episode manifests, long-form audio masters, or long-form rough cuts.
- Maximum `3` still passes for the same carrier.
- If motion exposes a source-baked still defect, route back to stills; do not use motion rerenders as anatomy repair.
- If a beat keeps failing for the same reason, change the carrier rather than endlessly tightening wording.
- If the motion proof cannot demonstrate contact-sheet-to-proof parity, route back before final export.

## Reset Rules

- Missing, waived-without-reason, transcript-mismatched, or question-like Signature Consequence Motif delivery reopens `script` or `audio` depending on the defect: missing/incorrect motif wording reopens `script`; correct wording with failed cadence reopens `audio`; caption-only motif mismatch reopens `video final`.
- Script changes reset audio and every downstream stage.
- Narration-map changes reset visual research, visual beatmap, shot plan, and every downstream visual stage.
- Audio transcript, cue-range, or script-content changes reset the narration map plus every downstream visual stage.
- Voice-only audio changes reset stills proof timing, motion proof timing, captions, and final export.
- Stills changes reset stills proof plus downstream motion stages for affected beats.
- Motion changes reset motion contact sheet, motion video proof, and final export.
- Production model decision changes reset visual beatmap, shot planning, render authorization, and downstream visual stages.
- Shot timing EDL changes reset motion video proof and final export.
- Caption changes reset final export only unless caption timing exposes a proof-level sync problem.
- Workflow scope manifest, DP research brief, visual research packet, visual beatmap, shot plan, or active ledger changes reset downstream visual stages for affected beats or shots.
- A mixed-review proof routes blockers back to stills or motion; it never becomes final.
